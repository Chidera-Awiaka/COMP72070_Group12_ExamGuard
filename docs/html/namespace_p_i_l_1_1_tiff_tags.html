◆ BYTE

int PIL.TiffTags.BYTE = 1

Map tag numbers to tag info.

id: (Name, Type, Length[, enum_values])

The length here differs from the length in the tiff spec. For numbers, the tiff spec is for the number of fields returned. We agree here. For string-like types, the tiff spec uses the length of field in bytes. In Pillow, we are using the number of expected fields, in general 1 for string-like types.
